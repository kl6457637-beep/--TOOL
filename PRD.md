# 亚马逊广告语义智推工作流 PRD (MVP)

## 产品概述

### 目的 & 背景

**目的**：设计并实现一套低门槛，强掌控、自动化的亚马逊广告优化工作流（MVP版本）。通过"硬性业务数据规则"与"大模型（LLM）语义推理"的解耦与结合，将单店铺日均广告优化耗时从 2 小时缩短至 10 分钟内，显著降低无效广告花费（ACOS），并最终自动生成符合亚马逊标准格式的 Bulk Upload 批量操作文件，实现业务闭环。

**背景**：当前亚马逊跨境电商赛道中，中小卖家普遍面临广告 ACOS 居高不下、无效流量浪费严重的问题。传统工具仅能基于基础数据（如点击量，花费）进行一刀切拦截，无法精准识别"语义不相关但产生花费"的隐性垃圾流量；全自动 AI 工具则因"黑盒操作"导致卖家严重缺乏信任感。市场急需一款"白盒化，带思考过程"的智能辅助决策工具。

---

### 用户画像与场景

**核心用户**：亚马逊中小卖家、跨境电商广告优化师（Ad Operator）。

**使用场景**：
- **日常报表巡检**（周/双周频次）：下载最新的 Search Term Report (STR，搜索词报表)，快速找出正在浪费预算的垃圾词并执行精准否定。
- **新品冷启动监控**（高频）：新品上架初期自动广告跑词阶段，利用语义排雷，防止预算跑到完全不相关的类目词上。
- **长尾词挖掘**：从杂乱的搜索词中找出低竞价、高转化且符合产品特定使用场景的长尾蓝海词，用于新建手动广告组。

---

### 需求范围与边界

#### 包含功能范围

本次 MVP 版本聚焦于核心工作流的跑通与单次会话（Session）体验，暂不接入亚马逊 SP-API 进行实时读写。包含以下范围：

- **前端交互端** (Web PC 网页)：上下文信息录入区、CSV 报表解析与上传、交互式诊断控制台（Dashboard）、批量操作表导出。
- **服务端与引擎区**：CSV 数据清洗脚本、硬规则判别引擎、LLM API 调度与并发控制模块（Prompt 组装与 JSON 解析）。
- **本地缓存** (Local Storage)：实现基础的用户操作记忆（如用户取消否定的"白名单词"本地缓存）。

#### 发布与商业化边界

- **发布策略**：独立 Web 工具形态，跟随 MVP 版本全量发布。
- **开放策略**：采用邀请制（Invite-Only）或白名单访问。MVP 阶段不强制接入支付网关，不限制单次上传的 CSV 行数（前端硬性限制最大 5 万行以防内存溢出），但需对单日 LLM API 调用次数进行并发限制。

#### 技术实现

- **LLM 集成**：支持千问（Qwen）API，用于语义分析
- **API 端点**：
  - `/api/analyze` - AI 语义建议（识别不相关词）
  - `/api/expand-keywords` - 场景化长尾拓词

#### 参考资料

- 参考竞品数据阈值：Helium 10 (Adtomic)、Jungle Scout (Cobalt)。
- 标准文件参考：Amazon Ads Sponsored Products Bulk Operations File Format (v3.0)。

### 需求详述

#### [REQ-01 & REQ-02] 数据输入与预处理引擎（P0）

**功能详述**

【序号1】**上下文录入区**（Context Input）：

- **交互**：提供两个多行文本框。一个是必填项 Listing 文本（建议用户粘贴 Title + 5 Bullets）；另一个是选填项 业务补充说明（PlaceHolder："例如：虽然是办公椅，但也适合孕妇久坐"）。
- **默认状态**：空，文本框失焦时进行非空校验（Listing部分）。

【序号2】**报表上传区**（Report Upload）：

- **交互**：拖拽或点击上传文件（支持 .csv, .xlsx），最大限制 20MB。
- **数据映射逻辑**：系统必须读取并映射以下核心列（若列名不完全匹配，需支持模糊正则匹配）：
  - Campaign Name（广告活动名称）
  - Ad Group Name（广告组名称）
  - Customer Search Term（客户搜索词）
  - Clicks（点击量）
  - Spend（花费）
  - 7 Day Total Sales（7天总销售额）
  - Total Return on Advertising Spend (ROAS) 或 ACOS

---

#### [REQ-03 & REQ-04 & REQ-05] 诊断建议控制台模块（P0）

**功能详述**

本页面为工具的核心工作台，采用多卡片式布局。

【序号1】**全局概览区**（Overview Metrics）：

- **计算逻辑**：
  - 分析总词数 = 上传表格中去重后的 Customer Search Term 总数。
  - 建议优化花费金额 = 被勾选建议否定的词的 Spend 总和。
- **交互反馈**：金额随下方列表用户的"勾选/取消"实时联动变化。

【序号2】**规则判定脱水区**（Rule-based Rejects）：

- **展示规则**：表格列表展示。默认全选（Checked）。
- **计算公式**（行业最佳实践假设，需提供设置入口）：
  - 条件 A（纯浪费）：Clicks >= 15 且 7 Day Total Sales == 0
  - 条件 B（严重亏损）：Clicks >= 5 且 ACOS > 80%（若表中无 ACOS 字段，通过 Spend / Sales 计算）
- **UI 字段**：广告词、所属广告组、点击量，花费、触发规则原因（如：高点无转）

【序号3】**AI 语义建议区**（Semantic Rejects）：

- **展示规则**：表格列表展示。默认全选（Checked）。
- **数据定义与 LLM Output 要求**：
  - 请求 LLM 返回 JSON 格式：`[{"keyword": "xxx", "action": "negative", "reason": "xxx"}]`
- **UI 字段增加一列** 【AI 诊断理由】。直接显示在表格中，无需 Hover。
- **交互反馈**：用户取消勾选某一行，触发 【REQ-07 记忆白名单】 逻辑，将该 Campaign + Keyword 写入浏览器 LocalStorage。

【序号4】**高潜长尾词区**（Potential Blue-Ocean）：

- **展示规则**：默认不勾选（Unchecked）。
- **计算逻辑**：Clicks >= 3 且 ACOS < 20% 且 LLM 判定 场景意图高度匹配 的长尾词（单词数 >= 3）

【序号5】**场景化长尾拓词**（Keyword Expansion）：

- **展示位置**：独立卡片区域，位于规则判定区上方
- **功能**：基于产品描述，调用 LLM 生成 20 个高转化长尾关键词
- **输出维度**：痛点维度、使用场景、目标人群、功能属性
- **交互**：点击按钮生成，支持复制单个或全部复制

#### [REQ-06] Bulk Upload CSV生成与导出模块（P0）

**功能详述**

【序号1】**导出控制面板**：

- **交互**：点击"一键导出优化包"按钮，弹出确认弹窗，显示："已选中 XX 个否定词，预计挽回无效花费 $XX。请选择匹配类型（下拉单选：精准否定 Negative Exact / 词组否定 Negative Phrase，默认选中 Negative Exact）"。

【序号2】**CSV 文件组装与编码生成**：

- **文件格式**：UTF-8 编码的 .xlsx 或 .csv（推荐 Excel 格式避免亚马逊乱码）。
- **数据映射规则**（必须严格遵守 Amazon V3.0 标准）：对于每一个勾选了"建议否定"的词，在导出表中生成一行数据。
- **生成列及固定值定义**：
  - **Product**：固定填入 Sponsored Products
  - **Entity**：固定填入 Negative Keyword（如果是词）或 Negative Product Targeting（如果是ASIN）
  - **Operation**：固定填入 Create
  - **Campaign Name**：取自原始上传报表对应行的 Campaign Name
  - **Ad Group Name**：取自原始上传报表对应行的 Ad Group Name
  - **Keyword Text**：取自原始报表的 Customer Search Term
  - **Match Type**：取用户在【序号1】中选择的类型（如 Negative Exact）
  - **State**：固定填入 Enabled

- **交互边界**：若用户一个词都没勾选，置灰导出按钮，Hover 提示"请至少选择一个优化项"。下载完成后，前端弹出 Toast："文件已生成，请直接前往亚马逊后台 -> 广告活动管理 -> 批量操作 进行上传"。
